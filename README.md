# kubectl-scheduler

> Warning! This codes are generated by Claude Code.

A read-only `kubectl` plugin that **simulates** whether a Pod could be scheduled onto a
specific Node, and—if not—**which filter blocks it and why**.

It runs the *actual* in-tree kube-scheduler Filter plugins (pinned to v1.32) against a
read-only snapshot of your cluster. It never creates, patches, binds, or changes anything.

## Install (krew, Apple Silicon macOS only)

Currently packaged for `darwin/arm64` only. Requires [krew](https://krew.sigs.k8s.io/).

Build the archive locally and install it without a release:

```
make dist                                    # produces dist/kubectl-schedule_v0.1.1_darwin_arm64.tar.gz
kubectl krew install \
  --manifest=plugins/schedule.yaml \
  --archive=dist/kubectl-schedule_v0.1.1_darwin_arm64.tar.gz
```

Or, once a matching GitHub release exists, install from the manifest URL:

```
kubectl krew install --manifest-url=https://github.com/alicek106/kubectl-scheduler/raw/main/plugins/schedule.yaml
```

Without krew, just drop the binary on your `PATH` as `kubectl-schedule`:

```
make build && cp dist/kubectl-schedule /usr/local/bin/
```

## Usage

```
kubectl schedule <pod> <node> [-n <namespace>] [--context <ctx>] [--no-color]
```

Uses your current kubeconfig context and namespace by default. Requires read (get/list)
access to pods, nodes, and (when the Pod uses PVCs) pv/pvc/storageclass/csinode.

## Example

```
$ kubectl schedule nginx worker-2

✗ BLOCKED   nginx → worker-2  (namespace: default)

Checked filters
  ✓ InterPodAffinity    ✓ NodeAffinity        ✓ NodeName
  ✓ NodePorts           ✓ NodeResourcesFit    ✓ PodTopologySpread
  ✗ TaintToleration   node had untolerated taint {dedicated=gpu:NoSchedule}

Not checked (1)
  ⊘ VolumeBinding   dynamic provisioning (WaitForFirstConsumer) decided at runtime

Summary  6 passed · 1 failed · 1 skipped
```

Result is one of: `✓ SCHEDULABLE`, `⊘ PASSED (with unchecked)`, or `✗ BLOCKED`.

## Limitations

This is a **best-effort simulation, not identical to your cluster's scheduler.** Specifically:

- **Filter stage only** — answers "does the Pod fit this node", not "would the scheduler pick
  this node". No scoring/prioritization and no preemption.
- **Pinned version** — scheduler logic is compiled at v1.32; a different cluster minor may differ
  (shown as a warning). The cluster's real `--feature-gates` cannot be read over RBAC.
- **Default profile only** — custom scheduler configs, extenders, and out-of-tree plugins are not
  reflected.
- **Unchecked filters are surfaced, never assumed to pass** — dynamic volume provisioning
  (WaitForFirstConsumer), DRA, and RBAC-restricted reads are reported as `SKIPPED`.
